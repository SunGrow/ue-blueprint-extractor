Architecture — Four Layers, Built Sequentially

Each layer is completed and tested before the next one starts

Built first
Layer 1 — Protocol
A formal specification built on JSON-RPC 2.0 — the same message format used by LSP and MCP. Defines every message type, the graph JSON schema, the type system (13 types with coercion rules), safety validation rules, execution lifecycle, and event streaming. This is the contract that every other layer implements.
JSON-RPC 2.0 Graph schema Type system Validation rules 80 conformance tests
▼ engine implements this contract ▼
Built second
Layer 2 — Engine
A standalone Rust binary. No UI. Each node runs as an independent Tokio async task with its own message channel. The scheduler runs nodes in parallel when their inputs are ready. Caches results by input signature so unchanged steps are skipped. Can run pipelines headlessly from the command line.
9 node types
LLM, Router, Sequence, Loop (×4), HITL, Command
2 LLM adapters
Claude Code CLI (native) + OpenRouter API (200+ models)
Safety system
Type checking, cost budgets, sandboxing, error ports
MCP server
13 tools for LLMs to create, edit, and run pipelines
▼ UI consumes the engine as a Rust library ▼
Built third
Layer 3 — Visual Editor
A Tauri 2 desktop app with React 19, Vite, and React Flow for the node canvas. Communicates with the engine via Tauri IPC. Shows streaming output per node in real time. Type-aware node palette, function subgraph navigation with breadcrumbs, per-iteration loop inspector.
React Flow v12 Zustand + Jotai Tauri 2 IPC TypeScript
Layer 4 — MCP API Surface
Embedded inside the engine. Lets any MCP-capable LLM create nodes, wire them, configure models, validate, and execute pipelines. Turns Pipeline Studio into programmable infrastructure.
Backflow updates
When a later layer reveals issues in an earlier one, the earlier layer is updated and its tests re-run. All changes are tracked in a protocol changelog.
