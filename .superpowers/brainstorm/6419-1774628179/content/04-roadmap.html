Implementation Roadmap — 3 Phases, 24 Milestones

Each phase completes fully before the next begins. Tests at every boundary.

P1
Phase 1: Protocol
Define the contract. No engine code, no UI code.
1.1 Message Schema
JSON Schema files for all protocol messages
1.2 Type System
13 types, compatibility matrix, wildcard resolution
1.3 Validation Rules
~30 graph fixtures testing every validation check
1.4 Conformance Suite
Rust test harness + ~80 JSON fixtures + mock server
Exit: Frozen protocol v0.1 — all fixtures pass against mock server
P2
Phase 2: Engine
Build the Rust engine. Headless execution without UI.
2.1 Data Model
Rust workspace (lib + bin crates), serde parsing, type checker
2.2 Validation Engine
All build-time checks, cycle detection, structured reports
2.3 Scheduler + Actors
Tokio actor per node, topological sort, parallel execution
2.4 Node Types
All 9 types including loops with condition re-pull
2.5 LLM Adapters
Claude Code CLI + OpenRouter API + mock adapter
2.6 Functions
Subgraphs, nesting, library loading, recursion detection
2.7 Caching
Input-signature hashing, LRU cache, skip unchanged nodes
2.8 Safety
Error port routing, cost budgets, process sandboxing
2.9 MCP Server
13 tools for programmatic graph manipulation via stdio
2.10 CLI Runner
Run pipelines from terminal: pipeline-studio run graph.json
Exit: All conformance tests pass against real engine. CLI can execute graphs. MCP server works.
P3
Phase 3: Visual Editor
Tauri 2 desktop app built on the proven engine.
3.1 Tauri Shell + IPC
Engine embedded as Rust library, IPC roundtrip <5ms
3.2 React Flow Canvas
Custom nodes, typed port colors, graph serialization
3.3 Wire Type Checking
Real-time validation, compatible ports highlight, tooltip errors
3.4 Node Palette
Searchable, type-filtered, right-click context menu
3.5 Config Panels
Per-node settings: model, prompt, budget, conditions
3.6 Live Streaming Output
Tokens appear in nodes as LLMs work, state indicators
3.7 Error Panel
Click error → canvas scrolls to the problem node
3.8 Function UI
Collapse to function, double-click to inspect, breadcrumbs
3.9 Loop Inspector
Pick an iteration number, see that pass's inputs and outputs
3.10 Project Save/Load
Git-friendly JSON files, autosave, recent projects
Exit: Full desktop app — create, configure, wire, run, debug, save, and reopen pipelines
