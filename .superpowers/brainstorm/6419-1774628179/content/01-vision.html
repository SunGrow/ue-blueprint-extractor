Pipeline Studio

A visual tool for building LLM workflows across projects, models, and tools

The Problem

Today, multi-step LLM work is manual. You copy output from one terminal, paste it into another, adjust prompts, re-run when something fails. Context is lost between steps. Errors are invisible until you check manually. Cost is untracked. Parallelism is impossible.

These aren't limitations of any individual LLM. They are limitations of treating LLM interactions as isolated conversations instead of steps in a structured workflow.

The Solution

Pipeline Studio is a desktop app where you build LLM workflows visually. Drag nodes onto a canvas, connect them with wires, run the whole pipeline with one click. Each node is a self-contained step: call an LLM, route data based on conditions, loop until quality is met, or pause for human review.

Typed data flow
Wires carry typed data (text, images, JSON). The system blocks you from connecting incompatible types before you run anything.
Any model, any CLI
Claude Code, Gemini, GPT, Kimi — each node can use a different model. CLI tools run natively with their full ecosystems (MCP, skills, agents).
Safe by default
Type checking at wire-time. Cost budgets on every LLM call. Process sandboxing. Error ports that force you to handle failures — or explicitly ignore them.
Reusable functions
Select a group of nodes, collapse to a function. Parameterized inputs and outputs. Share across projects via libraries.
Incremental execution
Change one step, re-run only what's downstream. Unchanged nodes use cached results. No wasted compute or API spend.
LLM-editable
A built-in MCP server lets LLMs create, modify, and run pipelines. The tool is programmable infrastructure, not just a GUI.
