Pipeline Studio — Architecture Overview

Protocol-first visual LLM workflow orchestrator — 42 features across 9 subsystems

System Architecture

PROTOCOL LAYER (JSON-RPC 2.0 + LSP-style capabilities)
Graph schema, node definitions, typed ports, execution states, safety rules — the single source of truth
7 protocol features Typed event stream Lifecycle management Cancellation Structured errors
ENGINE (Rust + Tokio)
Standalone headless binary
Reactive actor model (10 features) Tokio actor per node Topological scheduling Incremental caching Cycle detection + loop mode Pure/action distinction Node state machine
VISUAL EDITOR (Tauri + Web Canvas)
Separate app, connects to engine
Node canvas (5 UI features) Per-node live output Loop iteration history Error panel + jump-to-node Searchable node palette
EMBEDDED MCP SERVER
LLMs can edit the graph
Graph manipulation tools stdio transport Meta-orchestration

Safety System (Code Complete / Rust model)

Connection-time validation
Type mismatches caught when you wire
Build-time validation pass
Full graph validation before execution
Per-node error port
Every node has an error output — must be connected or explicitly ignored
Process sandboxing
CLI processes isolated, resource limits enforced
Cost budget enforcement
Per-node and per-pipeline token/cost limits with configurable action on breach
